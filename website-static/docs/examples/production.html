Production Setup

Best practices for using ReactScope in production environments.

Environment-Based Configuration

// main.tsx
import { ReactScopeProvider, createDashboardUIPlugin } from '@oxog/reactscope'

const isDev = import.meta.env.DEV
const isDebug = import.meta.env.VITE_DEBUG === 'true'

function App() {
  return (
    <ReactScopeProvider
      // Disable in production unless debugging
      enabled={isDev || isDebug}

      // Only load dashboard in development
      plugins={isDev ? [createDashboardUIPlugin()] : []}

      // Use sampling in production to reduce overhead
      options={{
        sampleRate: isDev ? 1 : 0.1, // 10% sampling in prod
        maxHistorySize: isDev ? 1000 : 100,
      }}
    >
      <YourApp />
    </ReactScopeProvider>
  )
}

Tree Shaking

ReactScope is fully tree-shakeable. Import only what you need: 

// Only imports useScope - other code is tree-shaken
import { useScope } from '@oxog/reactscope'

// Import specific plugins separately
import { createRenderTrackerPlugin } from '@oxog/reactscope/plugins/render-tracker'
import { createConsoleReporterPlugin } from '@oxog/reactscope/plugins/console-reporter'

Lazy Loading Plugins

import { ReactScopeProvider } from '@oxog/reactscope'
import { lazy, Suspense, useState } from 'react'

function App() {
  const [plugins, setPlugins] = useState([])

  // Load dashboard only when needed
  const loadDashboard = async () => {
    const { createDashboardUIPlugin } = await import('@oxog/reactscope')
    const plugin = createDashboardUIPlugin()
    setPlugins([plugin])
  }

  return (
    <ReactScopeProvider plugins={plugins}>
      <button onClick={loadDashboard}>
        Enable Profiler
      </button>
      <YourApp />
    </ReactScopeProvider>
  )
}

Performance Monitoring Service

import { createPlugin, ReactScopeProvider } from '@oxog/reactscope'

// Create a plugin that sends data to your monitoring service
const monitoringPlugin = createPlugin({
  name: 'monitoring',
  version: '1.0.0',

  hooks: {
    onRender: (event) => {
      // Only report slow renders
      if (event.duration > 50) {
        fetch('/api/metrics', {
          method: 'POST',
          body: JSON.stringify({
            type: 'slow_render',
            component: event.componentName,
            duration: event.duration,
            timestamp: Date.now()
          })
        })
      }
    }
  }
})

function App() {
  return (
    <ReactScopeProvider
      enabled={true}
      plugins={[monitoringPlugin]}
      options={{ sampleRate: 0.1 }}
    >
      <YourApp />
    </ReactScopeProvider>
  )
}

Conditional Tracking

import { useScope } from '@oxog/reactscope'

function ExpensiveComponent({ data }) {
  // Only track in development or when debugging
  const scope = useScope('ExpensiveComponent', {
    enabled: import.meta.env.DEV,
    trackProps: false // Disable props tracking for performance
  })

  return <div>{/* ... */}</div>
}

Recommendations

  • ✓ Use sampleRate to reduce overhead in production
  • ✓ Disable the dashboard UI in production builds
  • ✓ Use environment variables to toggle profiling
  • ✓ Only track critical components in production
  • ✓ Batch metric exports to reduce network calls
  • ⚠ Avoid trackProps on frequently updating components
